# Domain Docs

OpenBookmark is a single-context project.

## Before exploring or changing behavior

- Read the root `CONTEXT.md` and use its canonical vocabulary.
- Read the ADRs under `docs/adr/` that affect the area being changed.
- Surface any proposed behavior that contradicts an ADR instead of silently overriding it.

If a required term is absent from the glossary, note the gap and use domain modeling before inventing competing vocabulary.
