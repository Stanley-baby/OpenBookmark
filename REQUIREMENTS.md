# OpenBookmark 产品需求

**状态：** 已确认  
**日期：** 2026-07-21  
**首版平台：** Chrome、Edge（Manifest V3）

## 1. 产品目标

OpenBookmark 是一个免费、开源、local-first 的浏览器书签管理插件。它以独立品牌和净室代码，高还原度重现 Raindrop.io 的界面与核心工作流，同时提供用户可控、可恢复的自动备份。

首版必须在没有账号、没有服务器和没有网络连接的情况下完成全部本地书签管理操作。联网仅用于访问用户收藏的网页、抓取元数据以及执行用户配置的 WebDAV 备份。

## 2. 产品原则

- **数据归用户所有：** 本地数据是主数据，用户可随时导入、导出和恢复。
- **备份不等于同步：** 云盘只保存版本化备份；多设备同步使用独立机制。
- **独立实现：** CRX 仅用于识别视觉和行为，不复制其压缩源码、商标或原始素材。
- **隐私默认：** 不收集遥测，不持续监控浏览历史或页面内容。
- **先完成闭环：** 首版冻结明确边界，第二阶段功能不提前搭建。

## 3. 产品形态

插件包含两个主要界面：

1. **快速收藏弹窗：** 通过工具栏按钮、右键菜单或浏览器快捷键打开，保存或编辑当前页面。
2. **全页管理器：** 打开插件内置页面，管理全部书签、收藏夹、标签、回收站、迁移与备份设置。

首版不提供独立托管网站。界面支持中文和英文，默认跟随浏览器语言；首版只实现浅色主题。

## 4. MVP 功能范围

### 4.1 收藏

- 保存当前页面的 URL、标题、描述、封面来源、标签、备注和所属收藏夹。
- 支持编辑书签信息。
- 支持标记收藏和未读状态。
- 支持工具栏、右键菜单和可配置快捷键三个入口。
- 规范化 URL 后发现重复项时，默认打开已有书签，不创建副本。
- 不合并带有不同查询参数的 URL，避免把不同内容误判为重复项。

### 4.2 组织与浏览

- 创建、重命名、移动和删除嵌套收藏夹。
- 添加、移除和筛选标签。
- 按关键字段搜索、筛选和排序书签。
- 提供卡片视图和列表视图。
- 支持多选、批量移动和批量删除。
- 支持用户主动批量刷新所选书签的网页元数据。

### 4.3 删除与恢复

- 删除的书签先进入回收站，不立即永久删除。
- 回收站内容默认保留 30 天。
- 用户可恢复单项、永久删除单项或手动清空回收站。
- 删除记录必须保留足够信息，避免未来多设备同步时把已删除内容重新创建。

### 4.4 导入与导出

- 导入浏览器书签 HTML。
- 导入 Raindrop.io 的导出文件；具体格式在取得样本后确认映射。
- 导出通用浏览器书签 HTML。
- 导出包含全部领域数据的版本化 JSON。
- 导入和导出必须提供进度与明确的成功或失败结果。

### 4.5 封面与缓存

- 保存封面来源 URL 和用户自定义封面信息。
- 自动抓取的缩略图仅作为本地可再生缓存，不写入常规备份。
- 缩略图缓存默认上限为 500MB，超过上限时淘汰较少使用的缓存。

## 5. WebDAV 自动备份

### 5.1 备份边界

- 首版只支持 WebDAV；Google Drive 和 OneDrive 在第二阶段接入。
- 备份是单向、版本化的灾难恢复机制，不承担多设备同步。
- 备份包含完整书签领域数据、收藏夹结构、标签、设置和删除记录。
- 自动抓取的缩略图缓存不进入备份。

### 5.2 加密

- 默认在本地加密后上传，WebDAV 端只保存密文。
- 使用用户设置的恢复密码；项目无法找回遗失密码。
- 用户可明确关闭加密，以获得可直接读取的 JSON 备份。
- 本地书签数据库首版不加密。

### 5.3 调度与保留

- 数据发生变化后延迟 10 分钟触发自动备份。
- 同一小时最多自动备份一次。
- 默认保留最近 30 个备份版本。
- 插件启动时检查并补做错过的备份。
- 用户可随时执行手动备份。

### 5.4 恢复

- 先把备份导入临时数据库并验证格式与完整性。
- 恢复界面显示备份时间、版本、条目数量和差异摘要。
- 用户确认前不得修改当前数据。
- 确认恢复后整体替换当前数据库，不自动合并。
- 替换前自动备份当前数据库，为误操作保留回退路径。

## 6. 权限、隐私与安全

- 插件申请读取所有网站数据的权限。
- 即使具备该权限，也只在用户主动收藏或批量刷新元数据时读取页面。
- 不持续监控页面，不记录或上传浏览历史。
- WebDAV 连接仅访问用户配置的目标地址。
- 首版不包含任何使用统计、崩溃遥测或第三方分析 SDK。
- 错误诊断信息只保存在本地，由用户主动复制并提交。

## 7. 视觉与品牌

- 用户提供的 Raindrop.io CRX 是视觉与行为分析基准。
- 需要识别插件弹窗、全页管理器、设置页及其交互状态。
- 首版按浅色主题逐页进行截图对比验收。
- 不使用 Raindrop 名称、商标、图标、插画、原始素材或源码。
- 临时项目名称为 `OpenBookmark`；正式发布前另行完成重名与商标检索。

## 8. 数据规模与性能

- 设计容量上限为 100,000 条书签。
- 在常用桌面设备上，本地搜索、筛选和打开收藏夹的目标响应时间为 200ms 以内。
- 大批量导入、导出、备份和恢复允许耗时更长，但必须显示进度且不能阻塞界面。

## 9. 技术约束

- TypeScript
- React
- WXT
- Dexie / IndexedDB
- 浏览器原生 Web Crypto
- Chrome 与 Edge 共用 Manifest V3 代码
- AGPL-3.0 许可证

技术实现应优先复用浏览器原生能力和已选依赖，不为第二阶段预建抽象层。

## 10. 首版明确不做

- 深色主题
- Firefox 和 Safari
- 独立托管网站
- 网页永久副本或完整离线归档
- 团队协作和公开分享页
- 失效链接检测
- 独立的重复链接清理工具
- 网页正文全文检索
- AI 标签或 AI 分类
- Google Drive 和 OneDrive 备份
- 自托管同步服务器和多设备同步
- 本地数据库加密
- 遥测与分析

## 11. 后续阶段

### 第二阶段

- Google Drive 版本化备份
- OneDrive 版本化备份
- 深色主题
- Firefox 适配

### 第三阶段

- 可选的开源自托管同步服务器
- 多设备增量同步与冲突处理
- 根据真实使用反馈评估永久副本、全文检索和维护工具

Safari 是否支持，在核心体验稳定后单独评估，不作当前承诺。

## 12. 完成标准

MVP 只有在以下条件全部满足时才算完成：

- Chrome 和 Edge 均可侧载、启动和正常升级。
- 收藏、编辑、搜索、筛选、排序、批量操作和回收站流程通过。
- 浏览器书签与 Raindrop 数据可导入。
- HTML 与 JSON 导出可再次导入，并保持关键数据一致。
- WebDAV 加密备份、版本保留和恢复替换流程通过。
- 使用 100,000 条测试数据验证核心浏览操作。
- 键盘操作、焦点状态和基础无障碍检查通过。
- 主要界面的截图与 CRX 基准逐页对比并验收。
- 无默认遥测，无后台浏览记录采集。

## 13. 发布顺序

1. 发布 GitHub 源码、构建说明和可侧载 ZIP。
2. 用真实数据验证核心流程和备份恢复。
3. 修复阻断问题后提交 Chrome Web Store 与 Edge Add-ons。

## 14. 实施前输入

用户需提供 Raindrop.io 的 CRX 文件。取得文件后，先完成以下分析，不直接复制实现：

- Manifest、权限和插件入口清单
- 页面与状态清单
- 视觉尺寸、颜色、字体、间距和图标语义
- 交互流程和数据字段清单
- CRX 可见行为与本需求之间的差异清单

分析完成并确认没有阻断性缺口后，再开始编码。
